Class AbstractScraper

Abstract superclass for all "scrapers"

remarks

This class is very general - it is not locked down specifically to RYM. Each instance of a class extending AbstractScraper corresponds to one specific call - could be an api call or a simple GET call to retrieve a webpage body. The general use pattern for a subclass would be to first call the constructor, then AbstractScraper.scrape.

dataReadFromDB

dataReadFromDB: boolean
  • Defined in scrapers/abstractScraper.ts:37

Scrapers always check for a local copy of the target resource (using AbstractScraper.getEntity) before executing a scrape from an external resource. If the resource was found (and therefore no external calls made), this flag is set to true.

databaseID: number
